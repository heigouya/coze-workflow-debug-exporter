# Coze Workflow Debug Exporter

这是一个 Chrome 扩展 MVP，用来抓取 Coze / 扣子工作流试运行时前端接口返回的数据，并整理成适合发给 AI 排查问题的文本。

仓库首页的 `v0.1.0/`、`v0.1.1/`、`v0.1.2/`、`v0.1.3/`、`v0.1.4/`、`v0.1.5/`、`v0.1.6/`、`v0.1.7/`、`v0.2.0/`、`v0.2.1/`、`v0.2.2/`、`v0.2.3/` 文件夹分别保留了每个版本的完整代码；文件夹右侧的提交说明就是该版本的主要修改内容。

从 `v0.2.0` 起，调试报告页改成了「画廊 + 详情弹窗」的浏览方式：每个节点输出是一张带 JSON 高亮预览的卡片，采用 Masonry 瀑布流排列；右上角可切换「紧凑 / 默认 / 宽松」布局密度；点击卡片弹出 VS Code 深色风格的详情窗，可在「全部 / 输入 / 输出 / 错误」间切换、左右翻看相邻节点。抓包与导出逻辑保持不变。

## 安装方式

1. 打开 Chrome，进入 `chrome://extensions/`。
2. 打开右上角「开发者模式」。
3. 点击「加载已解压的扩展程序」。
4. 选择本文件夹，也就是 `coze-workflow-debugger-extension`。
5. 打开 Coze 工作流页面，重新试运行一次工作流。
6. 点击浏览器右上角扩展图标，选择「打开调试报告」。

## 怎么使用

- 报告页会按运行批次展示抓到的数据。
- 每张卡片代表一个节点或一个原始接口响应。
- 报错节点会用红色边框突出。
- 「复制本节点」只复制当前卡片。
- 「一键复制全部」会把本次运行的节点数据按顺序整理成一段文本，方便直接发给 AI。

## 当前 MVP 的限制

- 扩展只能抓到安装并启用之后，Coze 页面实际请求过的数据。
- 如果 Coze 页面没有主动请求某个节点详情，扩展也拿不到这个节点的详情；遇到这种情况，可以在 Coze 页面点开节点详情或重新试运行。
- 为避免浏览器消息传递被极端超长响应撑爆，单个接口响应正文超过约 8MB 会被截断。复制文本时默认会脱敏 token、secret、cookie、签名 URL 参数等明显敏感字段。
- 数据只保存在本机 Chrome 的扩展本地存储里，可以在弹窗中清空。

## 建议下一步

- 在真实 Coze 页面多跑几种工作流，确认哪些接口一定会返回完整节点数据。
- 如果需要“运行结束后、关闭页面再回来也能恢复历史”，下一版应优先接入 Coze 的运行历史接口，而不是只依赖前端抓包。
- 如果要公开发布到 Chrome Web Store，需要补隐私说明、权限解释、图标和更严格的数据存储策略。

## 0.1.1 更新

- 修复同一运行被重复展示的问题。
- 修复切换标签页后 Coze 自动刷新节点历史，导致同一节点重复增加的问题。
- JSON 展示改为彩色语法高亮。
- 输入、输出、错误信息区域都增加了独立复制按钮。

## 0.1.2 更新

- 区域复制控件改为图标按钮，只显示复制图标，不显示“复制输入/复制输出”等文字。
- 报告页节点标题优先显示 Coze 画布上的自定义节点名称，例如“视频抽帧”，不再优先显示数字节点 ID。
- 弹窗统计和报告分组改为按节点执行单元聚合，避免一次节点试运行被输入/输出记录拆成多次运行。

## 0.1.3 更新

- 复制图标改为更标准的“双矩形”复制图标，并紧跟在“输入/输出/错误信息”标题文字旁边。
- 节点名称采集会排除运行后的执行结果面板，不再把“运行成功”“1s”等状态文案当成节点标题。
- 运行统计继续按节点执行单元聚合，避免把节点本体和执行结果面板计为两个节点。

## 0.1.4 更新

- 进一步清洗拼接型节点标题，例如 `运行成功1s视频抽帧输入video输出...` 会显示为 `视频抽帧`。
- 对已缓存的错误节点标题也会在展示时二次清洗，不需要清空历史数据才能生效。
- 复制按钮改用用户指定的 SVG 图标，并按 18px 尺寸适配报告页。

## 0.1.5 更新

- 报告和统计改为优先基于 Coze 日志 Trace 接口：`list_spans` 和 `get_trace`。
- 运行次数按 `trace_id` / `log_id` 聚合，不再把节点详情接口单独算成一次运行。
- 节点卡片优先来自 Trace span，节点名优先使用 `node_name` / `alias_name` / `workflow_node_id`，减少 DOM 文本污染导致的乱码。
- `get_node_execute_history` 降级为补充数据，用于补齐单节点输入、输出和错误信息。

## 0.1.6 更新

- 弹窗统计改为聚焦当前工作流最新捕获的日志，不再显示本地历史总和。
- 切换 Coze 历史日志后，无需清空本地捕获，插件会把最新捕获日志作为当前日志。
- 报告页仍保留历史运行下拉，方便回看旧日志；默认使用最新捕获。

## 0.1.7 更新

- 运行记录下拉改为显示“工作流名称 / 运行时间”，不再直接展示 workflowId、traceId、logId 等机器 ID。
- 运行时间优先使用 Trace / 日志接口里的实际开始时间；拿不到时才显示插件捕获时间。
- “清空本地捕获”从弹窗移到报告页，并改名为“清空本地缓存”，避免在 Coze 工作流页误操作后影响继续观察。
- 报告页摘要增加本地累计日志数量，方便知道当前 Chrome 里积累了多少缓存。
