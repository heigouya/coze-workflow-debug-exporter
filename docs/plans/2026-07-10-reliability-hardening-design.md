# Coze 调试插件可靠性加固设计

## 目标

在不改变现有 Trace-first 主架构的前提下，保证接口数据不会因同时到达而丢失，复制内容能可靠脱敏，循环节点的每个批次都有入口可查看，并防止本地缓存无限增长。

## 数据处理方式

- `list_spans` / `get_trace` 继续作为一次运行的主数据。
- `get_node_execute_history` 继续补充节点详情。
- 后台把每次捕获排队保存，上一条写完后再处理下一条。
- 普通节点仍显示一张卡；带 `batch_index` 或 `sub_execute_id` 的记录存入该节点的 `iterations` 列表。
- 详情弹窗出现“批次”选择器，用户切换后查看对应输入、输出和错误。

## 隐私与缓存

- 复制时额外隐藏 cookie、session、普通 token、refresh token、API key 等字段。
- 缓存同时受“最多 500 条”和“约 40MB 文本数据”限制，优先移除最旧记录。
- popup 和报告页显示估算缓存大小。

## 验证方式

- 恢复开发阶段已有的 Trace、节点历史、名称映射和 UI 测试。
- 新增并发写入、循环批次、脱敏、大响应解析次数、状态颜色和刷新选择回退测试。
- 每个修复先让对应测试失败，再写最小修复让测试通过。
