# 循环体多次执行结果设计

## 问题

Coze 的 `get_trace` 返回同一个循环体节点的多条 span。它们的节点 ID 相同，但 `span_id` 不同，输入和输出也分别对应每一次执行。当前插件在整理数据时只按节点 ID 合并，因此只保留第一条 span 的输入和输出。

## 方案

保持报告里“一种节点只显示一张卡片”的现有布局。当同一运行中出现节点 ID 相同、`span_id` 不同的 trace 节点时，把每条 span 保存为该节点的一次执行记录 `iterations`。已有的批次选择器继续负责切换第 1、2、3……次的输入、输出和错误。

普通节点只有一条 span，不生成多次执行列表；trace 与节点历史接口对同一次执行的补充数据仍正常合并，不会被错误显示成循环。

## 验证

增加一个模拟 6 条 trace span 的自动测试，确认报告最终只有一张节点卡，但卡里保留 6 份不同的输入和输出。同时运行原有完整测试，确保显式 `batch_index` / `sub_execute_id` 批次和普通节点行为不变。
